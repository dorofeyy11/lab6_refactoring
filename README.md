# lab6_refactoring
## Рефакторинг кода

### Найденные проблемы качества
- Слишком длинная функция с несколькими ответственностями
- Повторяющаяся логика валидации
- Использование магических чисел
- Сложная вложенная структура условий

### Применённые рефакторинги
- Разделение логики на небольшие функции
- Вынос магических чисел в именованные константы
- Упрощение условий и циклов
- Применение принципов DRY и KISS

### Результат
Код стал короче, понятнее и легче для сопровождения.
Логика читается сверху вниз как сценарий.
Поведение программы сохранено, все тесты продолжают проходить.
